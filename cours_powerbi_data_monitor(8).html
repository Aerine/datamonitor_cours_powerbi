Introduction & setup
Power BI transforme des données brutes en dashboards interactifs sans écrire de code. On utilise la version web app.powerbi.com — aucune installation, fonctionne sur tous les OS.

Se connecter

Ouvrir app.powerbi.com et se connecter avec un compte Microsoft. Si pas de compte : créer un compte gratuit sur microsoft.com avec une adresse email professionnelle ou scolaire.

Fondamentaux 3 / 6
Power Query — nettoyer & enrichir
Power Query est l'éditeur de transformation intégré à Power BI. Toutes les modifications faites ici sont non destructives — le CSV original reste intact. Chaque transformation est enregistrée comme une étape qu'on peut annuler.

Étape 1 — Corriger les types prix et note

⚠ prix et note sont détectées comme texte (icône Aᵦ) car le CSV utilise un point décimal (849.00) et Power BI est configuré en français (virgule). La méthode standard échoue — voici la seule qui fonctionne.
1
Supprimer l'étape "Type de co..." automatique
Dans le volet Étapes appliquées → cliquer sur le × à gauche de l'étape "Type de co...". Les colonnes reviennent avec leur type brut.
2
Cliquer sur l'icône Aᵦ de la colonne prix
Cliquer directement sur l'icône Aᵦ dans l'en-tête de la colonne prix → choisir Utilisation des paramètres régionaux... (pas "Nombre décimal" seul — ça échoue).
3
Choisir Nombre décimal · Anglais (États-Unis)
Type de données = Nombre décimal · Paramètres régionaux = Anglais (États-Unis) → OK. L'icône passe en 1.2.
4
Répéter pour la colonne note
Même procédure sur la colonne note. Les autres colonnes (nb_avis, en_stock, nom, source, catégorie) sont déjà bien typées.

Étape 2 — Ajouter des colonnes calculées

L'onglet Ajouter une colonne → Colonne conditionnelle propose une interface graphique — pas besoin d'écrire de formule.

Colonne à créerRègleUtilité dans le dashboard
Segment prixprix ≥ 500 → "Premium" · prix ≥ 100 → "Mid-range" · sinon → "Entrée de gamme"Filtrer par segment dans un slicer
bien_notenote ≥ 4.5 → 1 · sinon → 0Compter les produits bien notés dans les mesures DAX
💡 Pourquoi Power Query plutôt qu'Excel ? Les transformations sont des étapes enregistrées. Quand le scraper tourne et génère de nouvelles données, Power BI les relit et rejoue automatiquement toutes les étapes — sans rien refaire. En Excel, il faudrait tout recommencer.

Étape 3 — Créer la colonne Segment prix

1
Ajouter une colonne → Colonne conditionnelle
Nommer la colonne Segment prix.
2
Configurer les règles
Règle 1 : prix est supérieur ou égal à 500 → sortie Premium. Ajouter clause → prix ≥ 100 → Mid-range. Sinon : Entrée de gamme.
3
Valider → OK
La colonne apparaît à droite. Vérifier quelques valeurs.

Étape 4 — Fermer et appliquer

Cliquer sur Créer un rapport en bas à droite (ou Accueil → Fermer et appliquer). Les données nettoyées et enrichies chargent dans Power BI.

Vue Modèle

Cliquer sur l'icône Modèle dans la barre de gauche du rapport. Chaque table importée apparaît comme une carte. On crée une relation en faisant glisser une colonne vers une autre.

Créer une relation entre deux tables

1
Glisser la colonne FK vers la PK
Ex : glisser categorie_id de la table produits vers id de la table categories. Une ligne de relation apparaît entre les deux cartes.
2
Vérifier la cardinalité
Double-clic sur la relation → vérifier Plusieurs à un (*:1). Sens du filtre : Single (la table "un" filtre la table "plusieurs").
3
Fusionner plusieurs CSV avec la même structure
Si chaque CSV correspond à une date de scraping différente → dans Power Query : Accueil → Nouvelle source → Dossier → sélectionner le dossier → Combiner. Power BI empile toutes les tables et ajoute une colonne Source.Name.

Mesures DAX
DAX (Data Analysis Expressions) permet de créer des calculs qui s'adaptent automatiquement aux filtres actifs dans le rapport. Une mesure DAX se recalcule à chaque clic — c'est ce qui rend le dashboard vraiment interactif.

Créer une mesure

Dans le volet Données à droite → clic droit sur la table → Nouvelle mesure → saisir la formule → Entrée.

Les mesures essentielles — avec équivalent Excel

MesureFormule DAXÉquivalent Excel
Prix moyenAVERAGE(produits[prix])MOYENNE(D:D)
Note moyenneAVERAGE(produits[note])MOYENNE(E:E)
Nb produitsCOUNTROWS(produits)NBVAL(A:A)
Nb bien notésCOUNTROWS(FILTER(produits, produits[note] >= 4.5))NB.SI(E:E;">=4.5")
Division sécuriséeDIVIDE(valeur_a, valeur_b, 0)=A/B sans gestion du /0

CALCULATE — calculer en ignorant les filtres du rapport

CALCULATE modifie le contexte de filtre. Quand un visuel affiche "Audio", toutes les mesures se calculent sur "Audio" seulement. CALCULATE permet de forcer un filtre spécifique — par exemple toujours calculer le prix moyen pour source = "Client", quel que soit ce que l'utilisateur a sélectionné.

DAX — CALCULATE pour Data Monitor
// Prix moyen Client — indépendant du filtre actif Prix moyen Client = CALCULATE(AVERAGE(produits[prix]), produits[source] = "Client") // Prix moyen Amazon Prix moyen Amazon = CALCULATE(AVERAGE(produits[prix]), produits[source] = "Amazon") // Écart en % — DIVIDE évite l'erreur si Amazon = 0 Écart % = DIVIDE([Prix moyen Client] - [Prix moyen Amazon], [Prix moyen Amazon]) * 100
Exercice ⏱ 10 min Créer tes premières mesures DAX

La prof a montré comment créer une mesure — à toi :

  1. Clic droit sur la table dans le volet Données → Nouvelle mesure
  2. Créer Prix moyen Client = CALCULATE(AVERAGE(produits[prix]), produits[source] = "Client")
  3. Créer Prix moyen Amazon = CALCULATE(AVERAGE(produits[prix]), produits[source] = "Amazon")
  4. Créer Écart % = DIVIDE([Prix moyen Client] - [Prix moyen Amazon], [Prix moyen Amazon]) * 100
  5. Ajouter une Carte KPI avec chaque mesure sur ton dashboard
  6. Activer un slicer sur source — les cartes CALCULATE changent-elles ? Pourquoi ?

Les cartes avec CALCULATE ne doivent PAS changer quand tu filtres sur "Client" dans le slicer — elles ignorent le filtre et calculent toujours sur source = "Client" (ou Amazon). C'est exactement le comportement voulu pour comparer. Si elles changent, vérifier la formule CALCULATE.

Calculer l'écart Client vs Amazon

Créer une mesure DAX pour l'écart de prix :

DAX — Mesure "Écart Client vs Amazon %"
Ecart Client vs Amazon % = DIVIDE( CALCULATE(AVERAGE(produits[prix]), produits[source] = "Client") - CALCULATE(AVERAGE(produits[prix]), produits[source] = "Amazon"), CALCULATE(AVERAGE(produits[prix]), produits[source] = "Amazon") ) * 100

DIVIDE gère la division par zéro. CALCULATE filtre sur une valeur spécifique de source.

Mise en forme & thème
Un dashboard sans mise en forme est illisible en soutenance. Cinq actions rapides qui font la différence.

1. Appliquer un thème

Affichage → Thèmes → choisir un thème clair et lisible. Eviter les fonds noirs pour les soutenances (projection souvent mauvaise). Le thème "Exécutif" ou "Innovateur" fonctionnent bien.

2. Titrer chaque visuel

Sélectionner un visuel → Format → Titre → activer → saisir une question : "Prix moyen par catégorie — Client vs concurrents". Un visuel sans titre est illisible hors contexte.

3. Cohérence des couleurs

Attribuer une couleur fixe par source dans tous les graphiques : Client = bleu, Amazon = orange, Fnac = vert, Darty = rouge. Clic droit sur une série → Couleur de la série.

4. Aligner et redimensionner

Sélectionner plusieurs visuels (Maj + clic) → Format → Aligner. Les visuels alignés donnent une impression de professionnalisme immédiate.

5. Ajouter une zone de texte titre de page

Insertion → Zone de texte → saisir le titre de la page : "Data Monitor — Analyse concurrentielle Mai 2025". Police grande, couleur de l'accent du thème.

→ Règle des 3 : pas plus de 3 couleurs distinctes, pas plus de 3 types de visuels différents par page, pas plus de 3 pages dans le rapport. La simplicité rend le dashboard mémorable.

Q
C'est quoi la différence entre Power BI et Tableau ?
Power BI est de Microsoft, s'intègre dans l'écosystème Office et domine en parts de marché. Tableau est plus puissant sur les très grosses volumétries et plus répandu dans les grandes entreprises data (banques, tech). Les deux s'apprennent facilement quand on en maîtrise un — mentionner les deux sur le CV est un plus.
Q
Différence entre mesure DAX et colonne Power Query ?
Une colonne Power Query est calculée une fois à l'import et stockée (statique). Une mesure DAX est recalculée dynamiquement à chaque interaction — clic sur un slicer, filtre de catégorie, changement de page. Règle pratique : colonne pour les segments et flags fixes, mesure pour tout ce qui s'agrège ou se filtre.
Q
J'ai plusieurs CSV (un par source scrappée) — comment les croiser dans Power BI ?
Importer chaque CSV séparément (Créer → CSV pour chacun). Ils apparaissent comme des tables distinctes dans le volet Données. Aller en vue Modèle → faire glisser le champ commun (ex : nom_produit) d'une table vers l'autre pour créer une relation. Les visuels peuvent ensuite croiser les deux tables automatiquement. Si les CSV ont la même structure (mêmes colonnes), l'approche plus simple est de les fusionner dans Power Query : Accueil → Nouvelle source → Dossier → pointer vers le dossier contenant tous les CSV → Power BI les empile en une seule table.
Q
Comment fusionner plusieurs CSV qui ont la même structure ?
Dans Power Query : Accueil → Nouvelle source → Dossier → sélectionner le dossier qui contient tous les CSV. Power BI liste les fichiers. Cliquer sur Combiner → Power BI empile toutes les tables et ajoute automatiquement une colonne Source.Name qui indique de quel fichier vient chaque ligne. Parfait si chaque CSV correspond à une date de scraping différente.
Q
Comment filtrer un visuel sans que ça affecte les autres ?
Sélectionner le visuel → dans le volet Filtres à gauche du canevas → Filtres sur ce visuel → glisser un champ et configurer la condition. Ces filtres s'appliquent uniquement au visuel sélectionné — contrairement aux Slicers qui filtrent toute la page.
Q
Un slicer ne filtre pas un visuel spécifique — comment faire ?
Sélectionner le slicer → onglet Format dans le ruban → Modifier les interactions. Une icône filtre/bloquer apparaît sur chaque visuel. Cliquer sur aucun (cercle barré) sur le visuel que tu veux exclure du filtre. Utile pour garder un KPI "Total général" fixe même quand un slicer est actif.
Q
Peut-on faire du Python dans Power BI ?
Oui — Power BI Desktop permet d'intégrer des scripts Python dans Power Query (pour transformer les données) et comme source de visuel. C'est un pont direct entre le scraper Python et la dataviz. Fonctionnalité avancée, pas utile pour la séance d'aujourd'hui, mais très valorisable en alternance.
Q
Comment connecter Power BI à une API REST ?
Dans Power Query : Obtenir des données → Web → saisir l'URL de l'API. Power BI fait un GET et parse automatiquement le JSON retourné. Si l'API nécessite des headers ou une authentification, cliquer sur Avancé pour les configurer. Utile si le projet expose une API Express plutôt qu'un CSV.
Q
Comment mettre à jour les données sans tout refaire ?
Dans Power BI Service : bouton Actualiser dans le menu du rapport. Power BI relit la source (CSV OneDrive, SQLite via ODBC…) et rejoue toutes les étapes Power Query. Tous les visuels se mettent à jour en quelques secondes sans rien reconfigurer.
Q
Mon graphique affiche "Somme de prix" à 42 000 — comment avoir la moyenne ?
C'est le comportement par défaut : Power BI somme les valeurs numériques. Cliquer sur la flèche ▼ à côté du champ dans la zone Valeurs → choisir Moyenne. Pour les prix et les notes, toujours choisir Moyenne. Pour les avis, Somme peut avoir du sens (volume total d'avis).
