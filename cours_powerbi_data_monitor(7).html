Introduction & setup
Power BI est l'outil de dataviz de Microsoft. Il transforme des données brutes en dashboards interactifs sans écrire de code. On utilise la version web app.powerbi.com — aucune installation nécessaire, fonctionne sur tous les OS.

Se connecter

Ouvrir app.powerbi.com et se connecter avec un compte Microsoft. Une fois connecté, la page d'accueil s'affiche.

Connecter les données
Power BI Service permet d'importer directement un fichier CSV depuis son ordinateur. Pour le projet complet, la connexion SQLite via Power BI Desktop offre une actualisation automatique à chaque nouveau scraping.
→ Pour la séance aujourd'hui — import direct du fichier CSV. Rapide, sans configuration, fonctionne sur tous les OS.

Pour la séance — Importer le CSV

1
Créer → (ou + Nouveau rapport depuis Accueil)
Cliquer sur Créer dans le panneau gauche. La page "Ajouter des données" s'affiche.
2
Cliquer sur la tuile CSV
Un explorateur de fichiers s'ouvre. Naviguer jusqu'à produits_export.csv et sélectionner le fichier.
3
Vérifier l'aperçu
Power BI affiche un aperçu des données. Vérifier que les colonnes sont bien détectées. Cocher "Utiliser la première ligne comme en-têtes" si nécessaire.
4
Créer un rapport vide ou automatique
Choisir "Créer un rapport vide" pour avoir la main sur les visuels. L'option "Générer un rapport prédéfini" crée automatiquement des visuels — utile pour explorer mais moins maîtrisé.
⚠ Ne pas déplacer ni renommer le fichier CSV après l'avoir importé — Power BI enregistre le chemin d'accès. Si le fichier est déplacé, il faudra le réimporter.

Pour le projet complet — Connexion SQLite via ODBC

Cette méthode nécessite Power BI Desktop (Windows). Elle connecte Power BI directement à la base SQLite — un clic Actualiser suffit après chaque scraping, sans réexporter de CSV.

Étape 1 — Installer le driver SQLite ODBC

1
Télécharger le driver
Aller sur ch-werner.de/sqliteodbc → télécharger sqliteodbc.exe (32 bits) ET sqliteodbc_w64.exe (64 bits) → installer les deux.
2
Vérifier l'installation
Ouvrir le menu Démarrer → rechercher "Sources de données ODBC" → onglet Pilotes → vérifier que SQLite3 ODBC Driver apparaît dans la liste.

Étape 2 — Configurer la source de données ODBC

3
Sources de données ODBC → Ajouter
Dans l'outil "Sources de données ODBC" → onglet DSN utilisateur → bouton Ajouter → choisir SQLite3 ODBC Driver → Terminer.
4
Configurer la source
Renseigner : Data Source Name = DataMonitor (nom libre) · Database Name = chemin complet vers le fichier .db (ex : C:\projets\data_monitor.db). Cliquer OK.

Étape 3 — Connecter Power BI Desktop à SQLite

5
Accueil → Obtenir des données → ODBC
Dans Power BI Desktop → Accueil → Obtenir des données → Plus → ODBC.
6
Choisir la source DataMonitor
Sélectionner la source ODBC créée à l'étape 4 (DataMonitor) dans le menu déroulant → OK.
7
Sélectionner les tables
Le navigateur affiche les tables de la base. Cocher les tables utiles (produits, categories…) → cliquer Transformer les données pour ouvrir Power Query.
8
Fermer et appliquer → Créer les visuels
Après nettoyage dans Power Query, Fermer et appliquer. Les données sont chargées — créer les visuels normalement.

Actualiser après chaque scraping

Quand le scraper Python tourne et met à jour la base SQLite, dans Power BI Desktop : Accueil → Actualiser. Tous les visuels se mettent à jour en quelques secondes.

⚠ Le chemin vers le fichier .db est absolu — si le fichier est déplacé, il faut mettre à jour la source dans Power Query → Source. Garder le fichier .db à un emplacement fixe pendant toute la durée du projet.

Alternative web — Via OneDrive (sans Power BI Desktop)

Si tu veux une actualisation automatique sans installer Power BI Desktop, faire exporter le CSV par le scraper directement dans un dossier OneDrive synchronisé. Power BI Service peut ensuite lire ce fichier et l'actualiser en un clic.

1
Modifier le scraper Python pour exporter dans OneDrive
Ajouter à la fin du script d'export :
df.to_csv(r"C:/Users/[nom]/OneDrive/data_monitor_export.csv", index=False, encoding="utf-8-sig")
Le dossier OneDrive est un dossier local normal — le scraper écrit dedans, OneDrive synchronise automatiquement vers le cloud.
2
Dans Power BI Service : Créer → tuile CSV → OneDrive
Sur app.powerbi.com → Créer → tuile CSV → choisir OneDrive comme emplacement → naviguer jusqu'au fichier data_monitor_export.csv.
3
Créer le rapport normalement
Power BI importe les données depuis OneDrive. Créer les visuels comme d'habitude.
4
Actualiser après chaque scraping
Quand le scraper tourne, il met à jour le CSV dans OneDrive. Dans Power BI Service : Actualiser dans le menu du rapport. Power BI relit le fichier OneDrive et met à jour tous les visuels.
→ Comparatif des méthodes pour le projet : OneDrive = 100% web, tous OS, nécessite que le scraper tourne sur une machine avec OneDrive installé. ODBC = plus direct, actualisation plus fiable, mais nécessite Power BI Desktop sur Windows.

Aller plus loin avec Power Query
Quand tu importes un CSV, Power BI ouvre automatiquement Power Query et applique 3 étapes : Source, En-têtes promus, Type de colonne. Dans la plupart des cas tout est déjà correct — tu n'as qu'à vérifier et cliquer Créer un rapport. Mais Power Query permet d'enrichir considérablement les données avant analyse.
✓ Ce que Power BI détecte tout seul sur le CSV Data Monitor : prix = décimal, note = décimal, nb_avis = entier, en_stock = entier, nom/source/catégorie = texte. Si un type est incorrect, deux méthodes pour le corriger — voir ci-dessous.

Onglet Ajouter une colonne — colonnes calculées

Deux façons d'ajouter une colonne calculée. La Colonne conditionnelle est recommandée pour les débutants — interface graphique sans formule.

Colonne conditionnelle ✓ recommandé
Interface graphique : choisir la colonne, l'opérateur, la valeur, le résultat. Génère automatiquement la formule if/else. Idéal pour créer Segment prix ou bien_note.
Ajouter une colonne → Colonne conditionnelle
Colonne personnalisée
Formule libre en langage M. Plus flexible mais nécessite de connaître la syntaxe : if [prix] >= 500 then "Premium" else "Standard"
Ajouter une colonne → Colonne personnalisée

Créer la colonne "Segment prix" avec Colonne conditionnelle

1
Ajouter une colonne → Colonne conditionnelle
La fenêtre de configuration s'ouvre.
2
Configurer la première règle
Nom de la nouvelle colonne : Segment prix. Colonne = prix · Opérateur = est supérieur ou égal à · Valeur = 500 · Sortie = Premium

Ajouter une clause "Sinon si"

Modèle de données
Si on a connecté plusieurs tables (ex : une table produits + une table catégories séparée), il faut créer une relation entre elles dans la vue Modèle. C'est le passage du MLD au moteur de Power BI.

Vue Modèle

Cliquer sur l'icône Modèle dans la barre de gauche. Chaque table importée apparaît comme une carte. On crée une relation en faisant glisser une colonne vers une autre.

Créer une relation

1
Faire glisser la colonne FK vers la PK
Ex : faire glisser categorie_id de la table produits vers id de la table categories. Une ligne de relation apparaît.
2
Vérifier la cardinalité
Double-clic sur la relation → vérifier que c'est bien Plusieurs à un (*:1) — plusieurs produits pour une catégorie. Sens du filtre : Single (la catégorie filtre les produits).
3
Vérifier dans un visuel
Retourner en vue Rapport et créer un graphique qui utilise un champ de chaque table — si la relation est correcte, les données se croisent automatiquement.
→ Pour Data Monitor avec un seul CSV, pas besoin de relations — tout est dans une table. La vue Modèle devient utile si vous avez exporté plusieurs CSV (ex : un par site scrappé) et que vous voulez les croiser.

Mesures DAX — les basiques

DAX (Data Analysis Expressions) est le langage de formules de Power BI. Pour l'instant, trois mesures suffisent :

MesureFormule DAXUsage
Prix moyenAVERAGE(produits[prix])KPI global ou par catégorie
Nb produitsCOUNTROWS(produits)Volume de catalogue
Note moyenneAVERAGE(produits[note])Qualité perçue

Créer une mesure : clic droit sur la table → Nouvelle mesure → saisir la formule.

Calculer l'écart Client vs Amazon

Créer une mesure DAX pour l'écart de prix :

DAX — Mesure "Écart Client vs Amazon %"
Ecart Client vs Amazon % = DIVIDE( CALCULATE(AVERAGE(produits[prix]), produits[source] = "Client") - CALCULATE(AVERAGE(produits[prix]), produits[source] = "Amazon"), CALCULATE(AVERAGE(produits[prix]), produits[source] = "Amazon") ) * 100

DIVIDE gère la division par zéro. CALCULATE filtre sur une valeur spécifique de source.

Mise en forme & thème
Un dashboard sans mise en forme est illisible en soutenance. Cinq actions rapides qui font la différence.

1. Appliquer un thème

Affichage → Thèmes → choisir un thème clair et lisible. Eviter les fonds noirs pour les soutenances (projection souvent mauvaise). Le thème "Exécutif" ou "Innovateur" fonctionnent bien.

2. Titrer chaque visuel

Sélectionner un visuel → Format → Titre → activer → saisir une question : "Prix moyen par catégorie — Client vs concurrents". Un visuel sans titre est illisible hors contexte.

3. Cohérence des couleurs

Attribuer une couleur fixe par source dans tous les graphiques : Client = bleu, Amazon = orange, Fnac = vert, Darty = rouge. Clic droit sur une série → Couleur de la série.

4. Aligner et redimensionner

Sélectionner plusieurs visuels (Maj + clic) → Format → Aligner. Les visuels alignés donnent une impression de professionnalisme immédiate.

5. Ajouter une zone de texte titre de page

Insertion → Zone de texte → saisir le titre de la page : "Data Monitor — Analyse concurrentielle Mai 2025". Police grande, couleur de l'accent du thème.

→ Règle des 3 : pas plus de 3 couleurs distinctes, pas plus de 3 types de visuels différents par page, pas plus de 3 pages dans le rapport. La simplicité rend le dashboard mémorable.
